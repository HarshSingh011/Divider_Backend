🏠 Home Screen - Authentication
GET /health
Check if the API server is running and healthy. No authentication required.
Response (200 OK)
{ "status": "OK" }
POST /auth/register
Create a new user account. Validates email format, password strength, and username format.
Request Body
email string *
Valid email address (e.g., user@example.com)
username string *
3-20 characters, letters/numbers/hyphens/underscores only
password string *
Min 8 chars: uppercase, lowercase, number, special char (@$!%*?&)
Request Example
curl -X POST https://divider-backend.onrender.com/auth/register \ -H "Content-Type: application/json" \ -d '{ "email": "john.doe@example.com", "username": "john_doe", "password": "SecurePass123!" }'
Response (201 Created)
{ "id": "usr_12345abcde", "email": "john.doe@example.com", "username": "john_doe", "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "created_at": "2026-04-01T10:30:00Z" }
Error Responses
400 Bad Request - Invalid email format { "error": "Invalid email format. Example: user@example.com" }
400 Bad Request - Weak password { "error": "Password must be at least 8 characters and contain uppercase, lowercase, number, and special character (@$!%*?&)" }
POST /auth/login
Authenticate user and receive JWT token for protected endpoints.
Request Body
email string *
Registered email address
password string *
Account password
Request Example
curl -X POST https://divider-backend.onrender.com/auth/login \ -H "Content-Type: application/json" \ -d '{ "email": "john.doe@example.com", "password": "SecurePass123!" }'
Response (200 OK)
{ "id": "usr_12345abcde", "email": "john.doe@example.com", "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "authenticated": true }
Error Response
401 Unauthorized - Invalid credentials { "error": "invalid email or password" }
💡 Save the token! Use it in the Authorization header for all protected endpoints:
Authorization: Bearer <your_token_here>
📊 Charts Screen - Candlestick (OHLC) Data
GET /trading/candles
Retrieve OHLC (Open, High, Low, Close) candlestick data for a specific stock symbol. Data is aggregated in 1-minute intervals.
Query Parameters
symbol string *
Stock symbol (e.g., RELIANCE-CE-2900, INFY-CE-1500)
limit integer
Number of candles to return (default: 50, max: 200)
Request Example
curl -X GET "https://divider-backend.onrender.com/trading/candles?symbol=RELIANCE-CE-2900&limit=10"
Response (200 OK)
[ { "symbol": "RELIANCE-CE-2900", "open": 45.20, "high": 47.80, "low": 44.90, "close": 47.50, "volume": 125000, "timestamp": "2026-04-01T10:35:00Z" }, { "symbol": "RELIANCE-CE-2900", "open": 47.50, "high": 49.10, "low": 47.30, "close": 48.90, "volume": 98500, "timestamp": "2026-04-01T10:36:00Z" } ]
✅ Display Tip: Use this data to render candlestick/bar charts on the Charts screen showing price movements over time.
💼 Portfolio Screen - Trading & Transactions
POST /trading/trade 🔒 REQUIRES AUTH
Execute a BUY or SELL trade. Updates wallet, records transaction in immutable ledger, and updates portfolio.
Request Body
symbol string *
Stock symbol (e.g., RELIANCE-CE-2900)
quantity number *
Number of units to buy/sell
price number *
Price per unit
type string *
Either "BUY" or "SELL"
Request Example
curl -X POST https://divider-backend.onrender.com/trading/trade \ -H "Content-Type: application/json" \ -H "Authorization: Bearer YOUR_JWT_TOKEN" \ -d '{ "symbol": "RELIANCE-CE-2900", "quantity": 10, "price": 45.50, "type": "BUY" }'
Response (200 OK)
{ "transaction_id": "txn_abc123def456", "symbol": "RELIANCE-CE-2900", "type": "BUY", "quantity": 10, "price": 45.50, "amount": 455.00, "fee": 2.275, "total": 457.275, "remaining_cash": 49542.725, "timestamp": "2026-04-01T10:30:00Z" }
📌 Fee Calculation: Automatically deducted at 0.5% of transaction amount (amount × 0.005)
POST /trading/deposit 🔒 REQUIRES AUTH
Deposit cash into wallet. Creates a deposit transaction in the immutable ledger.
Request Body
amount number *
Amount to deposit (in rupees)
Request Example
curl -X POST https://divider-backend.onrender.com/trading/deposit \ -H "Content-Type: application/json" \ -H "Authorization: Bearer YOUR_JWT_TOKEN" \ -d '{ "amount": 50000 }'
Response (200 OK)
{ "deposit_id": "dep_xyz789", "amount": 50000, "previous_balance": 0, "new_balance": 50000, "timestamp": "2026-04-01T10:25:00Z" }

🤝 Trading Operations - Buy & Sell Stocks
POST /trading/trade 🔒 REQUIRES AUTH
Buy Stock: Purchase shares of a stock at specified price. Validates sufficient available cash, deducts fee (0.5%), and updates portfolio.
Request Body
symbol string *
Stock symbol to buy (e.g., RELIANCE-CE-2900)
quantity number *
Number of shares to buy (must be > 0)
price number *
Price per share (must be > 0)
type string *
Must be "BUY"
BUY Request Example
curl -X POST https://divider-backend.onrender.com/trading/trade \ -H "Content-Type: application/json" \ -H "Authorization: Bearer YOUR_JWT_TOKEN" \ -d '{ "symbol": "RELIANCE-CE-2900", "quantity": 10, "price": 48.75, "type": "BUY" }'
Response (200 OK) - BUY
{ "status": "Trade executed successfully", "symbol": "RELIANCE-CE-2900", "type": "BUY", "quantity": 10, "price": 48.75, "total": 487.50, "transaction_id": "txn_abc123def456", "timestamp": "2026-04-01T10:35:00Z" }
📌 BUY Order Flow:
1. Validates available cash ≥ (quantity × price × 1.005)
2. Deducts amount from wallet
3. Creates transaction record
4. Updates portfolio with new holding
POST /trading/trade 🔒 REQUIRES AUTH
Sell Stock: Sell shares from your portfolio at specified price. Validates sufficient holdings, deducts fee (0.5%), and credits to wallet.
Request Body (Same as BUY)
symbol string *
Stock symbol to sell (e.g., INFY-CE-1500)
quantity number *
Number of shares to sell (must ≤ holdings)
price number *
Price per share
type string *
Must be "SELL"
SELL Request Example
curl -X POST https://divider-backend.onrender.com/trading/trade \ -H "Content-Type: application/json" \ -H "Authorization: Bearer YOUR_JWT_TOKEN" \ -d '{ "symbol": "INFY-CE-1500", "quantity": 5, "price": 63.50, "type": "SELL" }'
Response (200 OK) - SELL
{ "status": "Trade executed successfully", "symbol": "INFY-CE-1500", "type": "SELL", "quantity": 5, "price": 63.50, "total": 317.50, "transaction_id": "txn_xyz789abc123", "timestamp": "2026-04-01T10:36:00Z" }
📌 SELL Order Flow:
1. Validates sufficient holdings (quantity in portfolio)
2. Calculates net proceeds = (quantity × price) - fee
3. Adds net proceeds to wallet
4. Creates transaction record
5. Updates portfolio, removes if quantity reaches 0
Error Response (Insufficient Holdings)
400 Bad Request - Not enough holdings { "error": "insufficient holdings: need 20 units but only have 15" }
⚡ Real-Time Updates - WebSocket Advanced Guide
WS wss://divider-backend.onrender.com/ws 🔒 REQUIRES AUTH
WebSocket connection for real-time market data streaming. Connect with JWT token and receive live price updates every 500ms.
Connection Setup (JavaScript)
// Get JWT token from login const token = localStorage.getItem('token'); // Create WebSocket connection const ws = new WebSocket( `wss://divider-backend.onrender.com/ws?token=${token}` ); // Handle connection opened ws.onopen = () => { console.log('WebSocket connected'); }; // Handle incoming messages ws.onmessage = (event) => { try { const marketData = JSON.parse(event.data); console.log('Market update:', marketData); // Update UI with real-time prices marketData.forEach(stock => { updateStockDisplay(stock.symbol, stock.currentPrice); }); } catch (error) { console.error('Message parse error:', error); } }; // Handle errors ws.onerror = (error) => { console.error('WebSocket error:', error); // Attempt reconnect after delay setTimeout(() => connectWebSocket(), 5000); }; // Handle disconnection ws.onclose = () => { console.log('WebSocket disconnected'); // Auto-reconnect setTimeout(() => connectWebSocket(), 5000); };
Real-Time Data Format
[ { "symbol": "RELIANCE-CE-2900", "currentPrice": 48.75, "volume": 125000, "percentageChange": 4.84 }, { "symbol": "HDFC-PE-1400", "currentPrice": 38.90, "volume": 98500, "percentageChange": -2.75 }, { "symbol": "INFY-CE-1500", "currentPrice": 63.50, "volume": 112300, "percentageChange": -1.17 }, { "symbol": "TCS-PE-3500", "currentPrice": 102.30, "volume": 156200, "percentageChange": 3.53 } ]
Update Frequency
Interval 500ms
Market data updates sent every 500 milliseconds
Connection Timeout 60 seconds
Inactive connections closed after 60 seconds of no activity
Max Message Size 64KB
Maximum size of each WebSocket message frame
✅ Best Practices:
• Implement auto-reconnect with exponential backoff
• Parse JSON messages safely with try/catch
• Use throttling to prevent UI update flooding
• Keep connection alive with periodic pings
• Update only changed values for performance
⚠️ Error Codes & Troubleshooting
HTTP Status Codes
200 OK
Request successful
201 Created
Resource created successfully
400 Bad Request
Invalid request parameters or insufficient funds
401 Unauthorized
Missing or invalid JWT token
403 Forbidden
Authenticated but not authorized for this resource
404 Not Found
Resource or endpoint not found
500 Server Error
Internal server error - try again later
Common Error Messages
// Invalid credentials { "error": "invalid email or password" } // Token expired { "error": "token expired" } // Insufficient funds for BUY { "error": "insufficient funds: need ₹487.50 but have ₹250.00" } // Insufficient holdings for SELL { "error": "insufficient holdings: need 20 units but only have 15" } // Invalid stock symbol { "error": "invalid symbol" } // Weak password { "error": "Password must be at least 8 characters and contain uppercase, lowercase, number, and special character" }
Authentication Troubleshooting
401 Error on Protected Endpoint
✓ Check token is present in Authorization header
✓ Use format: "Authorization: Bearer YOUR_TOKEN"
✓ Verify token hasn't expired (24 hour limit)
✓ Try logging in again to get fresh token
CORS Errors
✓ All endpoints support CORS
✓ Ensure requests include proper headers
✓ Check Content-Type is application/json for POST
✓ WebSocket URL should use wss:// (not ws://)
